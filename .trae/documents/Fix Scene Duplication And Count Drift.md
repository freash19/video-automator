## Гипотеза причины (почему раньше не дублировалось)
- Дубли сцен начали появляться из‑за «двойных проверок» после вставки текста.
- Сейчас после клика по полю сцены идут дополнительные действия:
  - Пересохранение/повторная вставка в `_insert_text()` при несовпадении `inner_text` (доп. клики по `span_locator`, Tab и повторный ввод).
  - Пер-сценовая валидация `verify_scene_after_insert` через `_verify_scene_text()` с ретраями и вмешательством в фокус.
- В HeyGen UI сразу после клика/вставки происходит сдвиг/перерисовка, поэтому любые дополнительные клики/ретраи могут случайно активировать соседнюю сцену/плейсхолдер, визуально создавая «дубли» (как на 7/8).

## Цель
- Убрать проверки сразу после вставки текста и любые повторные «дотыкания»/переввод в рамках одной сцены.
- Оставить только финальную проверку после последней сцены (она уже есть в воркфлоу: reload + проверка наличия всех ожидаемых текстов).

## Изменения в коде
### 1) heygen_automation.py — отключить пер-сценовую проверку
- В `__init__` изменить дефолт `verify_scene_after_insert` на `False` (чтобы поведение было как раньше, если флаг не задан в config).
- В `fill_scene()` удалить/закомментировать блок `if self.verify_scene_after_insert: ... perform_step(verify_scene_...)`.
  - Либо оставить блок, но гарантировать, что по умолчанию он выключен (и включается только явной настройкой).

### 2) heygen_automation.py — убрать повторные перевставки в рамках одной сцены
- Внутри `_insert_text()` удалить блок, который читает `current_text` и делает до 2 повторных «click → Meta+A → Backspace → insert_text → Tab».
- В итоге `fill_scene()` будет строго:
  - один клик по полю
  - Meta+A → Backspace → insert_text → Tab
  - без дополнительных кликов и без повторов

### 3) Финальная проверка остаётся
- Ничего не менять в финальном шаге проверки после заполнения (reload + проверка наличия ожидаемых текстов), т.к. это и есть «единственная валидация» по твоему требованию.

## Верификация
- Прогнать существующий e2e (или CSV с проблемным кейсом), сравнить:
  - До фикса: появляются дубли/лишние сцены.
  - После фикса: не появляется дублирование, а финальная проверка после reload продолжает отлавливать реально не вставленные тексты.
- Добавить скрин списка сцен до/после заполнения (только для диагностики при ошибке).

## Ожидаемый результат
- UI больше не получает лишних повторных кликов/перевставок в момент сдвига интерфейса.
- Пропадают «дубли» сцен, вызванные повторными действиями после вставки.
- Контроль качества остаётся за финальной проверкой после последней сцены.
