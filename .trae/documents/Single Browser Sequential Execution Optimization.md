Я оптимизирую логику выполнения задач, чтобы использовать один экземпляр браузера и одну вкладку для последовательной обработки. Это сохранит сессию и предотвратит тормоза системы из-за множества процессов.

**План:**

1. **Настройка последовательного выполнения**:

   * Убедиться, что `max_concurrency` равно `1` в `config.json`.

   * Добавить/обновить настройку `close_browser_on_finish` в `config.json` на `false`, чтобы браузер не закрывался автоматически после каждой задачи.

2. **Изменение** **`ui/runner.py`**:

   * Обновить метод `run_many`: убрать закрытие браузера после каждой отдельной задачи.

   * Логику закрытия браузера перенести в самый конец выполнения всего пакета задач (или вообще убрать, чтобы браузер оставался открытым для продолжения работы).

   * Обеспечить переиспользование уже открытого экземпляра `runner.automation` и его подключения к браузеру.

3. **Изменение** **`heygen_automation.py`**:

   * Доработать `process_episode_part`: убедиться, что при переходе к следующему эпизоду скрипт использует *существующую* страницу (`self._page`) и просто переходит по новой ссылке (`page.goto`), а не создает новый контекст/вкладку.

4. **Уточнение** **`ui/api.py`**:

   * Проверить, что запуск задач через API корректно подхватывает текущее состояние раннера.

**Результат:**

* При запуске списка проектов они будут обрабатываться по очереди в **одном и том же** окне браузера.

* Скрипт будет просто переходить на новую ссылку шаблона в текущей вкладке.

* Новые процессы браузера создаваться не будут, что устранит лаги.

* Функция ручной остановки (убивание процесса) сохранится.
