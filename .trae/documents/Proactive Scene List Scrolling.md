## Почему сейчас всё равно пропускает
- Сейчас прокрутка происходит реактивно: только когда `text_X` не найден в DOM, мы пытаемся его «догнать» скроллом.
- При виртуализации списка сцен HeyGen может убрать нижние `text_X` из DOM заранее, и к моменту следующей сцены `text_{N}` ещё не подгружен — поэтому сцена считается «missing» и пропускается.
- Нужен проактивный скролл: заранее держать в видимой области (и в DOM) окно из нескольких ещё не заполненных плейсхолдеров.

## Цель
- Перед каждым кликом по сцене гарантировать, что в списке сцен видно минимум 4 незаполненных плейсхолдера `text_#` (включая текущий/следующие), чтобы виртуализатор не выкидывал нужные сцены из DOM.

## План изменений
### 1) Добавить проактивный скролл «окна незаполненных сцен»
- В `HeyGenAutomation` добавить метод вроде `_ensure_unfilled_window(page, current_scene_idx, min_unfilled_visible=4, max_steps=20)`.
- Метод делает только скролл (без кликов):
  - Находит scrollable-ancestor списка сцен (через ближайший scrollable parent от видимого `span[data-node-view-content-react]`).
  - Считает количество **видимых** плейсхолдеров `text_\d+` в пределах viewport этого контейнера.
  - Если видимых плейсхолдеров меньше `min_unfilled_visible`, докручивает вниз `scrollTop += delta` (и/или `mouse.wheel`) и пересчитывает, пока не станет достаточно или пока не достигнут лимит шагов.

### 2) Встроить это в `fill_scene` до клика по сцене
- В `fill_scene()` перед выбором `text_{scene_number}`:
  - Сначала обеспечить наличие `text_{scene_number}` (как сейчас: при `count==0` делаем догоняющий скролл).
  - Затем вызвать `_ensure_unfilled_window(..., current_scene_idx=scene_number, min_unfilled_visible=4)`.
  - После этого сделать единственный быстрый клик по центру поля и вставку текста.

### 3) Сделать параметр настраиваемым
- Добавить в конфиг параметр (по умолчанию 4), например `min_unfilled_scenes_visible=4`, чтобы можно было быстро подкрутить под разные шаблоны.

## Проверка
- Прогон на длинном CSV (10–15 сцен) на шаблоне с виртуализированным списком:
  - В логах не должно быть `scene_field_missing` после сцены 4–5.
  - Заполняются все сцены подряд без пропусков.

Если ок — начну вносить изменения по этому плану.
